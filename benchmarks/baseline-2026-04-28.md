# Baseline benchmarks вЂ” 2026-04-28 (M5 Max 128GB)

Updates [baseline-2026-04-26.md](baseline-2026-04-26.md) with the v0.3.0 dependency-chain refresh:

1. **mlx#19 retain commit landed in the local mlx submodule** вЂ” `MTL::Buffer` retain on first sighting in the bind path closes the long-standing `Invalid Resource` race at concurrent custom-kernel workloads.
2. **Removed redundant `stopGradient + asyncEval`** from `compressedAttention` вЂ” the swift-side band-aid we had only existed to mask the now-fixed underlying race. Removing it recovered ~10% throughput on TurboQuant MoE at Bв‰Ґ17.
3. **TurboQuant compressed-attention is now the default decode path** (`useCompressedAttention = true`).

The 2026-04-26 doc remains the date-locked snapshot of v0.2.2's measurements. This document captures the v0.3.0 candidate state.

## Methodology

- Platform: Apple M5 Max, 128 GB unified memory, macOS 26.x.
- Bridge: `vllm-swift` swift bridge built against alpha-tip `mlx-swift-lm` with retain commit cherry-picked into the `mlx` submodule.
- Env: `MLX_QUEUE_DEPTH=64 MLX_METAL_RETAIN_BOUND_BUFFERS=1`.
- 50 decode tokens for vllm-swift `bench_throughput.py` cells.
- Identical-prompts mode for short-context (matches README headline numbers).
- Single sample per cell. Median-of-3 still pending.

## InnocentVictim datapoint (long sustained sweep)

During the cap-redundancy sweep (Qwen3.5-35B-A3B B=16 8K turbo4v2, 10 runs back-to-back), one of ten runs returned `kIOGPUCommandBufferCallbackErrorInnocentVictim` вЂ” a GPU-wide reset (collateral discard) on a heated M5 Max, not a memory-corruption regression from the retain code. Documented: long sustained sweeps on M5 Max benefit from 30 s cooldowns between cells.
